# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
> Requirements in scope: AUTH-01, AUTH-02, AUTH-03, REST-01, REST-02

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS primitives only) |
| Preset | not applicable |
| Component library | none — hand-built with Tailwind |
| Icon library | Lucide React (`lucide-react`) |
| Font | Fira Sans — Google Fonts via `next/font/google` |

**Decision rationale:** No shadcn. The stack is Next.js 15 App Router with Tailwind. The brand palette is locked by Driva Dev's brand guide and Clerk's `appearance` prop wraps Clerk's own component renderer — shadcn would add unnecessary dependency overhead for Phase 1's three screens. Lucide React is chosen over Heroicons for its first-class TypeScript types and named import tree-shaking.

**Clerk appearance note:** Clerk's `<SignIn />` and `<SignUp />` components render their own UI. Use the `appearance` prop with `variables` and `elements` overrides to push brand colors through Clerk's theming API. Custom outer layout (background, card wrapper, footer) is fully controlled by our code.

---

## Tailwind Brand Tokens

These tokens are the canonical source. Every component must reference these — no raw hex values in JSX.

```typescript
// tailwind.config.ts — extend.colors
colors: {
  brand: {
    principal: '#EA580C',  // bg-brand-principal — primary buttons, CTAs
    titulares: '#9A3412',  // text-brand-titulares — H1, H2
    acento:    '#FED7AA',  // bg-brand-acento — badges, active nav bg, borders
    fondo:     '#FFF7ED',  // bg-brand-fondo — global background, page surface
    texto:     '#1C1917',  // text-brand-texto — all body text
  }
}
```

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Tailwind Class | px Value | Usage |
|-------|---------------|----------|-------|
| xs | `p-1` / `gap-1` | 4px | Icon-to-label gap, inline badge padding |
| sm | `p-2` / `gap-2` | 8px | Compact row spacing, icon button padding |
| md | `p-4` / `gap-4` | 16px | Default element spacing, form field gap |
| lg | `p-6` / `gap-6` | 24px | Card padding, section padding |
| xl | `p-8` / `gap-8` | 32px | Layout gaps, dashboard main content padding |
| 2xl | `p-12` / `gap-12` | 48px | Auth card vertical rhythm |
| 3xl | `p-16` / `gap-16` | 64px | Full-page vertical centering offset |

Exceptions:
- Touch targets for interactive elements: minimum `min-h-[44px] min-w-[44px]` (WCAG 2.5.5 AA)
- Sidebar width: fixed `w-64` (256px) — not on the 8pt scale by design; sidebar width is a layout constant
- Sidebar nav item height: `py-2.5 px-3` (10px/12px) for readable nav density at the 5-item list length

## Typography

All text uses Fira Sans loaded via `next/font/google`. No fallback font stack visible to users — `next/font` self-hosts and preloads.

| Role | Tailwind Classes | Size | Weight | Line Height | Color Token |
|------|-----------------|------|--------|-------------|-------------|
| H1 (page title) | `text-2xl font-bold` | 24px | 700 | 1.2 | `text-brand-titulares` |
| H2 (section heading) | `text-base font-bold` | 16px | 700 | 1.2 | `text-brand-titulares` |
| H3 (card heading) | `text-base font-medium` | 16px | 500 | 1.4 | `text-brand-titulares` |
| Body | `text-sm font-normal` | 14px | 400 | 1.5 | `text-brand-texto` |
| Label (form) | `text-sm font-medium` | 14px | 500 | 1.4 | `text-brand-texto` |
| Caption | `text-xs font-light` | 12px | 300 | 1.4 | `text-brand-texto` |
| Monospace (slug) | `font-mono text-sm` | 14px | 400 | 1.5 | `text-brand-principal` |

Minimum visible size on screen: 12px (caption/badge). No text below 12px anywhere.

**Type scale (4 sizes only):** 12px · 14px · 16px · 24px — H2 and H3 share 16px but are differentiated by weight (700 vs 500).

**Font weights in use:** 300 (light/caption), 400 (body), 500 (medium/H3/label), 700 (bold/H1/H2). Loaded via:
```typescript
Fira_Sans({ subsets: ['latin'], weight: ['300', '400', '500', '700'], variable: '--font-fira-sans' })
```

## Color

| Role | Hex | Tailwind | Usage |
|------|-----|----------|-------|
| Dominant (60%) | `#FFF7ED` | `bg-brand-fondo` | Global page background, auth page bg, dashboard bg |
| Secondary (30%) | `#FFFFFF` | `bg-white` | Sidebar bg, cards, auth card, input bg |
| Accent (10%) | `#FED7AA` | `bg-brand-acento` | Active nav item bg, badges, decorative borders |
| Heading text | `#9A3412` | `text-brand-titulares` | All H1/H2/H3, active nav text, badge text |
| Body text | `#1C1917` | `text-brand-texto` | Body, labels, secondary text |
| Interactive | `#EA580C` | `bg-brand-principal` | Primary buttons, input focus ring, links, slug display |
| Destructive | `#DC2626` | `text-red-600` / `border-red-500` | Error messages only — not used for any action in Phase 1 |

**Accent (`#FED7AA`) reserved exclusively for:**
- Active nav item background in sidebar
- Badges (e.g., "Próximamente" on disabled nav items)
- Divider/border-bottom between sidebar sections
- Slug confirmation card border

**Off-palette rule:** `bg-white` (`#FFFFFF`) is used as the secondary surface (cards, sidebar, auth card). It is not in the brand palette as a named token but is the natural card surface color against the `#FFF7ED` cream background. No other off-palette colors are permitted. No grays from Tailwind's default palette except `text-gray-400` for disabled nav text (see disabled state below).

**Shadows:** One allowed shadow only: `shadow-sm` (`0 1px 2px rgba(0,0,0,0.05)`) on cards. No `shadow-md`, `shadow-lg`, or custom shadows.

**Border radius:** All components use `rounded-lg` (8px). Inputs use `rounded-md` (6px). Pills/badges use `rounded-full`.

## Component Contracts

### 1. Auth Pages (Sign-in / Sign-up)

**Layout:** Full-page centered card. Single column. Background: `bg-brand-fondo`. No split layout.

**Page wrapper:**
```
min-h-screen bg-brand-fondo flex flex-col items-center justify-center px-4 py-12
```

**Card:**
```
bg-white rounded-lg shadow-sm w-full max-w-md px-8 py-10
```
- Width: `max-w-md` (448px max, fluid below)
- Padding: `px-8 py-10` (32px horizontal, 40px vertical)
- No border (shadow-sm provides separation from fondo background)

**Card internal layout (our wrapper, above Clerk's component):**
```
Header section: mb-8
  H1: text-2xl font-bold text-brand-titulares mb-1
  Subheading: text-sm font-normal text-brand-texto

Clerk component: [SignIn or SignUp rendered here]

Cross-link: mt-6 text-center text-sm text-brand-texto
  Link: text-brand-principal hover:underline font-medium

Footer: mt-8 pt-6 border-t border-brand-acento text-center
  Text: text-xs font-light text-brand-texto
  "Desarrollado por Driva Dev"
```

**Clerk `appearance` prop (applied to both SignIn and SignUp):**
```typescript
appearance={{
  variables: {
    colorPrimary: '#EA580C',
    colorBackground: '#FFFFFF',
    colorText: '#1C1917',
    colorTextSecondary: '#1C1917',
    colorInputBackground: '#FFFFFF',
    colorInputText: '#1C1917',
    borderRadius: '6px',
    fontFamily: 'var(--font-fira-sans), sans-serif',
    fontSize: '14px',
  },
  elements: {
    card: 'shadow-none p-0',
    headerTitle: 'hidden',        // We render our own H1
    headerSubtitle: 'hidden',     // We render our own subheading
    footerAction: 'hidden',       // We render our own cross-link
    formButtonPrimary:
      'bg-brand-principal hover:bg-[#C2410C] text-white font-medium rounded-lg ' +
      'min-h-[44px] transition-colors duration-150',
    formFieldInput:
      'border border-gray-200 rounded-md focus:border-brand-principal ' +
      'focus:ring-1 focus:ring-brand-principal outline-none',
    formFieldLabel: 'text-sm font-medium text-brand-texto',
  },
}}
```

**Error display (Clerk-rendered):** Clerk renders inline field errors and banner errors. Our `appearance.elements` ensures they use our font/size. No custom error UI needed in Phase 1 — Clerk handles email taken, wrong password, etc.

## Responsiveness

Phase 1 targets desktop-first (dashboard is a management tool). Minimum supported viewport: 1024px wide.

Auth pages are responsive down to 375px (mobile) because users may register on phones.

**Auth page breakpoints:**
- Mobile (`< 640px`): Full-width card with `px-4` outer padding, `px-6 py-8` card padding
- Tablet+ (`>= 640px`): `max-w-md` centered card with standard `px-8 py-10` padding

**Dashboard layout:** No mobile nav in Phase 1. The sidebar is always visible. A responsive nav is deferred to Phase 4 (Polish & Brand).
