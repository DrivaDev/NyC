**Step 2 — Create app/api/sign-cloudinary-params/route.ts:**
This route signs Cloudinary upload params so the client can POST files directly to Cloudinary without ever seeing CLOUDINARY_API_SECRET.

```typescript
import { v2 as cloudinary } from 'cloudinary'

// Configure once at module scope — not inside the request handler
cloudinary.config({
  cloud_name: process.env.NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME,
  api_key:    process.env.NEXT_PUBLIC_CLOUDINARY_API_KEY,
  api_secret: process.env.CLOUDINARY_API_SECRET,
})

export async function POST() {
  const timestamp = Math.round(Date.now() / 1000)
  const paramsToSign = { timestamp, folder: 'menu-digital' }

  const signature = cloudinary.utils.api_sign_request(
    paramsToSign,
    process.env.CLOUDINARY_API_SECRET as string
  )

  return Response.json({
    signature,
    timestamp,
    api_key:    process.env.NEXT_PUBLIC_CLOUDINARY_API_KEY,
    cloud_name: process.env.NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME,
  })
}
```

CRITICAL: No auth check on this route. The signing is safe because (a) the signature is time-limited to 1 hour, (b) it scopes uploads to the 'menu-digital' folder only, (c) the API secret never leaves the server.
Do NOT add `import { auth } from '@clerk/nextjs/server'` here — Clerk auth is not needed.

**Step 1 — Create components/dashboard/DashboardHeader.tsx:**
This is a Client Component that reads `usePathname()` and renders the correct title for each dashboard route. It replaces the static `<header>` element in the layout.

```typescript
'use client'

import { usePathname } from 'next/navigation'

const titles: Record<string, string> = {
  '/dashboard':            'Dashboard',
  '/dashboard/categories': 'Categorías',
  '/dashboard/dishes':     'Platos',
  '/dashboard/qr':         'Mi QR',
}

export default function DashboardHeader() {
  const pathname = usePathname()
  const title = titles[pathname] ?? 'Dashboard'

  return (
    <header className="h-14 bg-white border-b border-brand-acento flex items-center px-6 justify-between shrink-0">
      <h1 className="text-base font-bold text-brand-titulares">{title}</h1>
    </header>
  )
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/sign-cloudinary-params | Unauthenticated POST requesting upload signature |
| Server → Cloudinary API | Server uses API secret to compute HMAC signature |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | app/api/sign-cloudinary-params/route.ts | mitigate | `CLOUDINARY_API_SECRET` read only from `process.env` server-side; never in `NEXT_PUBLIC_` vars; never returned in response body |
| T-02-01-02 | Elevation of Privilege | app/api/sign-cloudinary-params/route.ts | accept | Signature is time-limited (1 hour) and scoped to folder 'menu-digital'; worst case: anyone can upload an image to that folder — no PII risk, low-value target; cost is Cloudinary storage quota |
| T-02-01-03 | Denial of Service | app/api/sign-cloudinary-params/route.ts | accept | No rate limiting in Phase 2; route has no DB calls so impact is minimal; Vercel function invocation limits provide implicit throttle |
| T-02-01-04 | Spoofing | components/dashboard/DashboardHeader.tsx | accept | `usePathname()` reads the actual Next.js router state — cannot be spoofed by the user; no auth decision is made based on the title |
</threat_model>
