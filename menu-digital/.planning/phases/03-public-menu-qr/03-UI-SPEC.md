## Spacing Scale

Declared values (multiples of 4 only — Tailwind v4 default scale):

| Token | px value | Tailwind class | Usage in this phase |
|-------|----------|----------------|---------------------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps inside allergen badge row, divider insets |
| sm | 8px | `gap-2`, `p-2` | Gap between allergen badges, dish meta spacing |
| md | 16px | `px-4`, `gap-4` | Page horizontal padding, tab button horizontal padding, dish row padding |
| lg | 24px | `py-6`, `gap-6` | Category section vertical padding, restaurant header vertical padding |
| xl | 32px | `py-8` | Top/bottom margin on restaurant header |
| 2xl | 48px | `py-12` | `scroll-mt-12` — section anchor offset below sticky tab bar |
| 3xl | 64px | `pb-16` | Footer bottom breathing room |

Exceptions:
- Dish thumbnail: 80px square (`w-20 h-20`) — nearest multiple is 80px (5 × 16px). Not a standard token but falls on the 8-point scale.
- Allergen badge circle: 24px (`w-6 h-6`) — 6 × 4px. On scale.
- Sticky tab bar height: ~48px (`py-3` top+bottom = 12px each + text ~24px). Use `scroll-mt-12` (48px) on section anchors.
- Touch target minimum: 44px — satisfied by tab buttons (`px-4 py-3` = 12px top + 12px bottom + 20px text ≈ 44px tall).

## Component Contracts

### 1. RestaurantHeader

**Location:** Top of `/menu/[slug]` page, above the sticky tab bar. Server-rendered.

**Structure:**
```
<header class="bg-brand-fondo px-4 py-8">
  [logo — if logoUrl present]
  <img> — 80×80px rounded-full, object-cover, border border-brand-acento
  [if no logo — omit the img entirely, no placeholder logo]
  <h1 class="text-2xl font-bold text-brand-titulares leading-tight mt-4">
    {restaurant.name}
  </h1>
  [if description present]
  <p class="text-sm font-normal text-brand-texto leading-normal mt-2 max-w-prose">
    {restaurant.description}
  </p>
  [if no description — omit the <p> entirely, no placeholder text]
</header>
```

**Rules:**
- Logo size: 80×80px (`w-20 h-20`), `rounded-full`, `object-cover`, thin `border border-brand-acento`
- If `logoUrl` is empty string or null — skip the `<img>` entirely. No placeholder icon, no empty circle.
- If `description` is empty string or null — skip the `<p>` entirely. Header shows name only.
- Name is always present (required field on Restaurant model).
- Max width on description: `max-w-prose` (~65 characters) for readability.

### 2. MenuCategoryNav

**Location:** Sticky below restaurant header. Only `'use client'` island on the page.

**Sticky behavior:**
- `sticky top-0 z-10` — sticks to viewport top the moment the restaurant header scrolls past.
- Background: `bg-white` (not brand-fondo) to visually separate from scrolling content.
- Bottom border: `border-b border-gray-100` — subtle 1px divider.

**Tab bar structure:**
```
<nav class="sticky top-0 z-10 bg-white border-b border-gray-100 overflow-x-auto">
  <div class="flex gap-0 px-4">
    [for each category]
    <button
      class="px-4 py-3 text-sm font-medium whitespace-nowrap border-b-2 transition-colors duration-150
             [active]   border-brand-principal text-brand-principal
             [inactive] border-transparent text-brand-texto hover:text-brand-titulares"
    >
      {category.name}
    </button>
  </div>
</nav>
```

**Overflow behavior:**
- `overflow-x-auto` — horizontal scroll on mobile when categories overflow viewport width.
- `whitespace-nowrap` on each button — prevents tab labels from wrapping.
- No scrollbar styling unless needed (native mobile scrollbar is acceptable).

**Active tab state:**
- Active: `border-brand-principal text-brand-principal border-b-2`
- Inactive: `border-transparent text-brand-texto` with hover `hover:text-brand-titulares`
- Transition: `transition-colors duration-150`

**IntersectionObserver spec:**
- One observer per category section.
- `rootMargin: '-20% 0px -70% 0px'` — active tab switches when section enters upper 30% of viewport.
- `threshold: 0`
- Initial active: first category by default (set via `useState(categories[0]?._id ?? null)`).
- Cleanup: `observers.forEach(o => o.disconnect())` on unmount.

**Scroll-to-section on click:**
```js
el?.scrollIntoView({ behavior: 'smooth', block: 'start' })
```
No `scroll-behavior: smooth` on `<html>` required. `scrollIntoView` is scoped and sufficient.

- Thin `border-b border-gray-100` below the category header, before the first dish.

### 4. DishRow

**Location:** Within a CategorySection. Server-rendered.

**Structure:**
```
<article class="flex items-start gap-4 px-4 py-4 border-b border-gray-100 last:border-b-0">
  <!-- Left: image slot (fixed 80×80px) -->
  <div class="relative w-20 h-20 shrink-0 rounded-md overflow-hidden bg-gray-100">
    [if dish.imageUrl]
      <Image src={dish.imageUrl} alt={dish.name} fill sizes="80px" class="object-cover" />
    [else]
      <DishPlaceholder />   ← see ImagePlaceholder contract
    [/if]
  </div>

  <!-- Right: text stack -->
  <div class="flex-1 min-w-0">
    <p class="text-base font-medium text-brand-texto leading-normal">{dish.name}</p>
    [if dish.description]
      <p class="text-sm font-normal text-brand-texto leading-normal mt-1 line-clamp-2">
        {dish.description}
      </p>
    [/if]
    <p class="text-base font-bold text-brand-titulares mt-1">
      {price formatted as ARS currency}
    </p>
    [if dish.allergens.length > 0]
      <div class="flex items-center gap-1 flex-wrap mt-2">
        [AllergenBadge × n]
      </div>
    [/if]
  </div>
</article>
```

**Price formatting:**
```ts
(dish.price / 100).toLocaleString('es-AR', { style: 'currency', currency: 'ARS' })
```
Price is stored as centavos integer. This renders as `$1.500,00` in Argentine locale.

**Dividers:**
- `border-b border-gray-100` on each `<article>`, with `last:border-b-0` to avoid double border at section bottom.

**Text truncation:**
- Dish description: `line-clamp-2` — shows at most 2 lines on mobile, prevents tall rows for long descriptions.

### 5. AllergenBadge

**Location:** Inside DishRow allergen area. Server-rendered (no interactivity state needed — CSS-only tooltip).

**Structure:**
```
<span class="relative group/badge" role="img" aria-label="{allergenLabel}">
  <!-- Circle -->
  <span class="flex items-center justify-center w-6 h-6 rounded-full bg-brand-acento text-sm select-none cursor-default">
    {emoji}
  </span>
  <!-- Tooltip -->
  <span
    class="absolute bottom-full left-1/2 -translate-x-1/2 mb-1.5 px-2 py-1
           bg-brand-titulares text-white text-xs font-normal rounded whitespace-nowrap
           opacity-0 group-hover/badge:opacity-100 pointer-events-none
           transition-opacity duration-150 z-20"
    aria-hidden="true"
  >
    {allergenLabel}
  </span>
</span>
```

**Allergen emoji mapping** (from D-11 — SINGLE SOURCE OF TRUTH is `lib/allergenEmoji.ts` to be created):

| Key | Emoji |
|-----|-------|
| gluten | 🌾 |
| crustaceos | 🦐 |
| huevos | 🥚 |
| pescado | 🐟 |
| cacahuetes | 🥜 |
| soja | 🫘 |
| lacteos | 🥛 |
| frutos_de_cascara | 🌰 |
| apio | 🥬 |
| mostaza | 🌻 |
| sesamo | 🫙 |
| dioxido_de_azufre | 🍷 |
| altramuces | 🌼 |
| moluscos | 🦪 |

**Tooltip positioning:** Above the badge (`bottom-full`), centered horizontally (`left-1/2 -translate-x-1/2`), 6px gap (`mb-1.5`).
**Tooltip background:** `bg-brand-titulares` (`#9A3412`) with `text-white`.
**Tooltip visibility:** `opacity-0` default, `group-hover/badge:opacity-100` on hover. CSS-only — no JS.
**Mobile tap behavior:** CSS hover does not fire on touch. The `group/badge` approach means a tap focuses the element — add `focus-within:opacity-100` to the tooltip `<span>` for mobile tap support.

Updated tooltip class with mobile support:
```
opacity-0 group-hover/badge:opacity-100 group-focus-within/badge:opacity-100
```

**Accessibility:** `role="img"` + `aria-label="{allergenLabel}"` on the outer span. Tooltip span has `aria-hidden="true"` (redundant to aria-label).

**Rule from D-10:** Render the allergen row ONLY when `dish.allergens.length > 0`. If empty, omit the `<div>` entirely.

## Interaction Specs

### Sticky Tab Bar

| Trigger | Behavior |
|---------|----------|
| Page load | Tab bar at natural position below restaurant header |
| Restaurant header scrolls out of viewport | Tab bar sticks to `top: 0` via CSS `position: sticky` |
| Viewport scroll back up to header | Tab bar releases from sticky, returns to natural flow |
| Category section enters upper 30% of viewport | Corresponding tab becomes active (IntersectionObserver fires) |
| User taps/clicks a tab | Smooth scroll to section anchor; active tab updates immediately on click (do not wait for IntersectionObserver) |

### Scroll Behavior

- Mechanism: `scrollIntoView({ behavior: 'smooth', block: 'start' })`
- Section offset: `scroll-mt-12` (48px) on every `<section id="category-...">` — prevents tab bar from covering section header
- No `scroll-behavior: smooth` on `<html>` element (scoped API call is sufficient and avoids side effects)

### IntersectionObserver Active Tab

- `rootMargin: '-20% 0px -70% 0px'` — section activates when it enters the top 30% of the viewport
- `threshold: 0` — fires as soon as any pixel of the section crosses the margin
- Initial state: first category is active by default
- On tab click: update `activeId` immediately in state (optimistic — do not wait for observer callback)
- Cleanup: disconnect all observers on component unmount

### Allergen Tooltip

| Platform | Trigger | Show | Hide |
|----------|---------|------|------|
| Desktop | Hover mouse over badge | `group-hover/badge:opacity-100` | Mouse leaves badge |
| Mobile | Tap badge (focus-within) | `group-focus-within/badge:opacity-100` | Tap elsewhere |

Tooltip appears above the badge. No animation except opacity transition (`duration-150`). No JS required.

### Settings Textarea

- No character counter UI — `maxLength={200}` enforces limit natively.
- No autosave — user submits via existing "Guardar cambios" button.
- Disabled during form submission (`pending` state from `useActionState`).
- No rich text, no markdown preview.

## Responsive Breakpoints

Mobile-first. The public menu is primarily a mobile experience (QR scan on phone).

| Breakpoint | Tailwind prefix | What changes |
|------------|----------------|--------------|
| Base (< 640px) | none | Full design as specified. Tab bar scrolls horizontally if many categories. Dish thumbnail 80×80px. Line-clamp-2 on dish description. |
| sm (≥ 640px) | `sm:` | Max-width container (`sm:max-w-lg sm:mx-auto`) — page content centered on tablet. Restaurant header padding increases (`sm:px-8 sm:py-10`). |
| md (≥ 768px) | `md:` | Desktop: `md:max-w-2xl`. Dish thumbnail can increase to 96px (`md:w-24 md:h-24`). Tab bar tabs gain more horizontal padding (`md:px-6`). |
| lg+ | `lg:` | No further breakpoints needed — content is inherently single-column. Max-width container caps width. |

**Tab bar on mobile:** `overflow-x-auto` with `whitespace-nowrap` on tabs. Horizontally scrollable — no wrapping to second row.

**No layout changes between breakpoints** — horizontal list row layout is used at all sizes. No card grid on desktop.
