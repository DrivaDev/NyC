## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Button icon-to-label gap, nav item icon-to-text gap (mr-3 = 12px exception below) |
| md | 16px | Card internal padding base, input horizontal padding |
| lg | 24px | Card padding (p-6), section gap between cards |
| xl | 32px | Page-level flex gap |
| 2xl | 48px | Empty state top/bottom padding (py-12) |
| 3xl | 64px | Full empty state vertical centering (py-16) |

Exceptions:
- Nav item icon-to-text: `mr-3` (12px) — established pattern in Sidebar, do not change.
- Touch targets: minimum 44px height on all interactive elements (`min-h-[44px]`). Existing pattern; apply to landing CTA buttons.
- Inline delete confirm buttons: `min-h-[32px]` — smaller because they appear inline in a row, not as standalone CTAs. This exception is intentional.
- Sidebar width: 256px (`w-64`) — fixed, not a spacing token, do not change.
- Drawer z-index: `z-40` for overlay backdrop, `z-50` for drawer panel and all floating elements (toasts).

## Typography

Font: **Fira Sans** — loaded via `next/font/google`, applied as `--font-sans` in `@theme`.

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Display / H1 (landing) | 28px | Bold (700) | 1.2 | `text-2xl font-bold` |
| Heading / H1 (page titles inside admin) | 24px | Bold (700) | 1.2 | `text-2xl font-bold` |
| Subheading / H2 (card headers) | 16px | Bold (700) | 1.2 | `text-base font-bold` |
| Body | 14px | Regular (400) | 1.5 | `text-sm font-normal` |
| Label / UI control | 14px | Medium (500) | 1.4 | `text-sm font-medium` |
| Caption | 12px | Light (300) | 1.4 | `text-xs font-light` |

Rules:
- H1 and H2 always use `text-brand-titulares` (#9A3412). Source: BRAND-02, CLAUDE.md.
- Body text always uses `text-brand-texto` (#1C1917).
- Minimum rendered size: 12px (caption). No text below 12px anywhere.
- Landing page product wordmark: Display (28px Bold) in `text-brand-titulares`.

## Color

Brand palette — all tokens defined in `app/globals.css`:

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-brand-fondo` | #FFF7ED | Page backgrounds (`bg-brand-fondo`) |
| Secondary (30%) | white / `--color-brand-acento` | #FFFFFF / #FED7AA | Card surfaces (`bg-white`), sidebar bg (`bg-white`), nav borders (`border-brand-acento`) |
| Accent (10%) | `--color-brand-principal` | #EA580C | Primary buttons, focus rings, spinner, active link bg not applicable |
| Heading tint | `--color-brand-titulares` | #9A3412 | H1, H2, active nav item text, badge text, `text-brand-titulares` |
| Accent surface | `--color-brand-acento` | #FED7AA | Badges, nav active bg, card borders, dividers, secondary backgrounds |
| Body text | `--color-brand-texto` | #1C1917 | All body/label/caption text |
| Danger | `--color-brand-danger` | #DC2626 | Delete buttons, error toasts, error text, required field asterisk — ONLY |

Accent (`--color-brand-principal`) reserved for:
- Primary button background (`bg-brand-principal`)
- Primary button focus ring (`focus:ring-brand-principal`)
- Loading spinner (`text-brand-principal`)
- Input focus border (`focus:ring-brand-principal` / `focus:border-brand-principal`)
- Secondary button border (`border-brand-principal`) and text (`text-brand-principal`)

Danger (`--color-brand-danger`) reserved for:
- Delete / trash icon button border and icon (`border-brand-danger`, `text-brand-danger`)
- Delete button hover background (`hover:bg-brand-danger/10`)
- Destructive confirm button background (`bg-brand-danger`, `hover:bg-[#B91C1C]`)
- Inline delete confirmation label text (`text-brand-danger`)
- Error toast border (`border-brand-danger`)
- Error toast icon (`text-brand-danger`)
- Form error message text (`text-brand-danger`)
- Required field asterisk (`text-brand-danger`)
- Upload error text (`text-brand-danger`)

**Never use:** gradients, `shadow-*` beyond `shadow-sm`, off-palette colors. Source: BRAND-06.

### 2. Mobile Drawer (Admin Layout)

**Source decisions:** D-MOB-01, D-MOB-02, D-MOB-03, D-MOB-04

Architecture: Extract a `DashboardShell` client component that owns `sidebarOpen` state and renders `DashboardHeader` + `Sidebar`. `app/(admin)/layout.tsx` becomes a server component that renders `DashboardShell` with `restaurantName`.

**Hamburger button** (inside `DashboardHeader`, mobile only):

```
<!-- Visible only on mobile: md:hidden -->
<button
  onClick={() => onOpenSidebar()}
  aria-label="Abrir menú"
  class="md:hidden flex items-center justify-center w-10 h-10 rounded-lg hover:bg-brand-fondo transition-colors duration-100 -ml-2"
>
  <Menu size={20} class="text-brand-titulares" />
</button>
```

Placement: Top-left inside `DashboardHeader`, before the page title. Icon: `Menu` (lucide-react, 20px).

**Drawer overlay:**

```
<!-- Backdrop: visible only when open, mobile only -->
<div
  class="fixed inset-0 bg-black/40 z-40 md:hidden"
  onClick={() => setOpen(false)}
  aria-hidden="true"
/>
```

- Opacity: `bg-black/40` (40% black).
- Z-index: `z-40`.
- Click anywhere on backdrop closes the drawer.

**Drawer panel:**

```
<aside class="fixed inset-y-0 left-0 z-50 w-64 md:hidden transform transition-transform duration-300 ease-in-out
              {open ? 'translate-x-0' : '-translate-x-full'}">
  <Sidebar restaurantName={restaurantName} onNavigate={() => setOpen(false)} />
</aside>
```

- Width: 256px (`w-64`) — same as desktop sidebar.
- Animation: CSS transform `translate-x-0` / `-translate-x-full` with `transition-transform duration-300 ease-in-out`.
- Z-index: `z-50` (above overlay at z-40).
- On mobile, `Sidebar` is rendered TWICE in the DOM: once in the drawer (mobile), once in the normal layout position (desktop). Use `md:flex` / `md:hidden` on the static sidebar to show/hide.

**Desktop sidebar** (unchanged):

```
<!-- Hidden on mobile, shown on desktop -->
<aside class="hidden md:flex w-64 ...">
  <Sidebar restaurantName={restaurantName} />
</aside>
```

**Sidebar `onNavigate` callback:**

Each `Link` in `Sidebar.tsx` gets `onClick={() => onNavigate?.()}`. The `onNavigate?: () => void` prop is optional — desktop usage passes nothing; drawer usage passes `() => setOpen(false)`.

**State location:** `DashboardShell` (new client component) owns `const [sidebarOpen, setSidebarOpen] = useState(false)`.

Z-index stack:
- Page content: default (no z-index)
- Toast notifications: `z-50`
- Drawer overlay: `z-40`
- Drawer panel: `z-50`
- Modals: `z-50`

## Interaction Contracts

### Button States

| State | Primary Button | Secondary Button | Danger Button |
|-------|---------------|-----------------|---------------|
| Default | `bg-brand-principal text-white` | `border-brand-principal text-brand-principal bg-transparent` | `bg-brand-danger text-white` |
| Hover | `hover:bg-[#C2410C]` | `hover:bg-brand-fondo` | `hover:bg-[#B91C1C]` |
| Focus | `focus:ring-2 focus:ring-brand-principal focus:ring-offset-2` | `focus:ring-2 focus:ring-brand-principal focus:ring-offset-2` | `focus:ring-2 focus:ring-brand-danger focus:ring-offset-2` |
| Disabled | `disabled:opacity-50 disabled:cursor-not-allowed` | same | same |
| Min height | `min-h-[44px]` | `min-h-[44px]` | `min-h-[44px]` (full CTA) / `min-h-[32px]` (inline confirm) |

### Drawer Animation

- Open: `translate-x-0` (slides in from left)
- Closed: `-translate-x-full` (hidden off-screen left)
- Transition: `transition-transform duration-300 ease-in-out`
- Overlay fade: `transition-opacity duration-300` (opacity 0 → `bg-black/40`)
- The drawer renders in DOM always (CSS transform controls visibility) — no conditional rendering that would cause layout shift.

### Toast Notification

- Position: `fixed bottom-6 right-6 z-50`
- Success: `border-brand-acento` border, `CheckCircle2` icon in `text-brand-principal`
- Error: `border-brand-danger/30` border, `XCircle` icon in `text-brand-danger`
- Auto-dismiss: 4000ms
- Min width: 280px, max width: `max-w-xs`

### Input Focus

- All inputs: `focus:outline-none focus:ring-2 focus:ring-brand-principal focus:border-brand-principal` — existing pattern, confirm preserved.
