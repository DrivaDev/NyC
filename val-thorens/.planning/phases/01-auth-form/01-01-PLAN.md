2. Create `app/api/auth/[...nextauth]/route.ts` with EXACTLY this content — do not deviate from this pattern (per CLAUDE.md critical constraint):
   ```typescript
   import NextAuth from "next-auth";
   import GoogleProvider from "next-auth/providers/google";

   const handler = NextAuth({
     providers: [
       GoogleProvider({
         clientId: process.env.GOOGLE_CLIENT_ID!,
         clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
         authorization: {
           params: {
             scope: "openid email profile https://www.googleapis.com/auth/gmail.send",
           },
         },
       }),
     ],
     callbacks: {
       async jwt({ token, account }) {
         if (account) {
           token.access_token = account.access_token;
         }
         return token;
       },
       async session({ session, token }) {
         session.access_token = token.access_token as string;
         return session;
       },
     },
   });

   export { handler as GET, handler as POST };
   ```

   CRITICAL: The scope string MUST include `https://www.googleapis.com/auth/gmail.send`. NEVER request broader Gmail scopes (e.g., gmail.readonly, mail.google.com). This is a hard constraint from CLAUDE.md.

3. Create `types/next-auth.d.ts` with EXACTLY this content:
   ```typescript
   import NextAuth from "next-auth";

   declare module "next-auth" {
     interface Session {
       access_token?: string;
     }
   }

   declare module "next-auth/jwt" {
     interface JWT {
       access_token?: string;
     }
   }
   ```

   This augmentation is required so TypeScript does not error when accessing `session.access_token` in downstream API routes.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → NextAuth | OAuth flow initiation — user clicks sign in |
| Google OAuth → NextAuth callback | Google returns authorization code + access_token |
| NextAuth JWT → session | access_token stored in encrypted JWT cookie |
| Session → API routes | Downstream routes read access_token from session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | NextAuth scope config | mitigate | Scope locked to `openid email profile https://www.googleapis.com/auth/gmail.send` — never expanded. Grep gate on route.ts must not find any broader Gmail scope. |
| T-01-02 | Information Disclosure | access_token in JWT cookie | mitigate | NextAuth encrypts JWT with NEXTAUTH_SECRET. Secret sourced from env var (never hardcoded). NEXTAUTH_SECRET must be ≥32 random bytes. |
| T-01-03 | Spoofing | Google OAuth callback | accept | Handled by NextAuth v4 — state parameter CSRF protection is built in. No additional mitigation needed for this phase. |
| T-01-04 | Tampering | tailwind.config.ts custom colors | accept | Config file — no user input. Low-value target. Build-time only. |
| T-01-05 | Information Disclosure | .env.example committed to repo | accept | .env.example contains no secrets (all values blank). .env (with real secrets) must be in .gitignore — verify .gitignore contains `.env` before first commit. |
</threat_model>
