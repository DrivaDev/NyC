The complete component structure:
```typescript
export default function Home() {
  const { data: session, status } = useSession();
  const [view, setView] = useState<"login" | "form" | "loading">("login");
  const [signingIn, setSigningIn] = useState(false);

  if (status === "loading") return null;

  // Authenticated → show form or loading
  if (status === "authenticated") {
    if (view === "loading") {
      return <LoadingView />;
    }
    return <FormView session={session} onSubmitComplete={() => setView("loading")} />;
  }

  // Unauthenticated → login
  return <LoginView signingIn={signingIn} onSignIn={() => { setSigningIn(true); signIn('google'); }} />;
}
```

Implement LoginView, FormView (shell only), and LoadingView as separate named functions in the same file (not exported) to keep the file organized and make PLAN-03 additions clean.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated browser → page.tsx | User should only see login screen — no form data accessible |
| NextAuth session cookie → useSession() | Client reads session; token is server-encrypted |
| Client-side view state | "loading" / "form" state controlled client-side only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | app/page.tsx auth gate | mitigate | Auth gate uses `status === "authenticated"` from NextAuth — form never renders when unauthenticated. Grep: page.tsx must contain `status === "unauthenticated"` or `status !== "authenticated"` guard before rendering FormView. |
| T-02-02 | Information Disclosure | session.user.email in DOM | accept | Email displayed in form header per AUTH-04 requirement — user's own email, intentional display. |
| T-02-03 | Spoofing | signIn redirect during loading | accept | signIn('google') is NextAuth-managed — CSRF state param handled by NextAuth. No additional mitigation. |
| T-02-04 | Denial of Service | Button double-click during OAuth redirect | mitigate | `signingIn` boolean state disables button after first click with `disabled` attribute and `cursor-not-allowed` class. |
</threat_model>
