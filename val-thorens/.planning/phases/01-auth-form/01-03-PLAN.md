function validate(data: FormData): FormErrors {
  const errs: FormErrors = {};
  if (!data.name.trim()) errs.name = "El nombre es obligatorio";
  if (!data.cv) errs.cv = "Solo se aceptan archivos PDF de hasta 5 MB";
  if (data.jobTypes.length === 0) errs.jobTypes = "Selecciona al menos un tipo de trabajo";
  if (!data.languages.trim()) errs.languages = "Indica al menos un idioma";
  if (!data.availFrom || !data.availTo) errs.dates = "Indica las fechas de disponibilidad";
  return errs;
}

**PDF upload handler** (validates file type and size, converts to Base64):
```typescript
function handleFile(file: File) {
  if (file.type !== "application/pdf" || file.size > 5 * 1024 * 1024) {
    setErrors(e => ({ ...e, cv: "Solo se aceptan archivos PDF de hasta 5 MB" }));
    return;
  }
  const reader = new FileReader();
  reader.onload = (e) => {
    const base64 = (e.target?.result as string).split(",")[1];
    setFormData(d => ({ ...d, cv: file, cvBase64: base64 }));
    setErrors(e => ({ ...e, cv: undefined }));
  };
  reader.readAsDataURL(file);
}
```

**handleSubmit function** (add inside FormView):
```typescript
async function handleSubmit(e: React.FormEvent) {
  e.preventDefault();
  const errs = validate(formData);
  setErrors(errs);
  if (Object.keys(errs).length > 0) return;

  setIsSubmitting(true);
  try {
    fetch("/api/run", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        name: formData.name,
        cvBase64: formData.cvBase64,
        jobTypes: formData.jobTypes,
        languages: formData.languages,
        availFrom: formData.availFrom,
        availTo: formData.availTo,
        accessToken: session.access_token,
      }),
    }).catch(() => {
      // Phase 1: /api/run does not exist yet — ignore network error
      // Phase 2 will implement the route; error handling added in Phase 3
    });
    onSubmitComplete();
  } catch {
    setErrors(e => ({ ...e, submit: "Error al iniciar el proceso. Intentalo de nuevo." }));
    setIsSubmitting(false);
  }
}
```

NOTE on fetch pattern: In Phase 1, /api/run does not exist, so the fetch will fail. We call `onSubmitComplete()` immediately (not awaiting the fetch result) to transition to loading state. Phase 3 will wire up SSE response handling. The `.catch()` on the fetch suppresses the console error from the missing route. This is intentional — the transition to loading state is the correct Phase 1 behavior per ROADMAP.md Phase 1 scope.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File input → client memory | PDF file read into browser memory via FileReader — never sent to server raw |
| Client form → POST /api/run | JSON payload crosses to server — includes cvBase64, accessToken |
| Client-side validation | 5MB + PDF type check before any processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | File upload — PDF validation bypass | mitigate | Client-side: validates `file.type === "application/pdf"` AND `file.size <= 5 * 1024 * 1024`. Phase 2 MUST re-validate server-side before passing to Puppeteer. Note in PLAN-03 SUMMARY for Phase 2 planner. |
| T-03-02 | Information Disclosure | cvBase64 in POST body | accept | CV sent to own server API (same origin), used in-memory during pipeline run only (CLAUDE.md constraint). Never persisted. Server-side enforcement in Phase 2. |
| T-03-03 | Elevation of Privilege | accessToken in POST body | mitigate | Token sent over HTTPS (Vercel enforces HTTPS in production). Token is user's own OAuth token scoped to gmail.send only. Server must not log it. Phase 2 note: never log accessToken. |
| T-03-04 | Denial of Service | Large file submission | mitigate | 5MB client-side check prevents large payloads before fetch. Server-side limit enforced by Vercel's 4.5MB body limit as additional backstop. |
| T-03-05 | Spoofing | CSRF on form submit | accept | Form calls same-origin /api/run only. No cross-origin mutation endpoint in Phase 1. NextAuth session cookie is HttpOnly — not accessible to attacker JS. CSRF risk is low for same-origin API. |
</threat_model>
